# History

This is the only default route to records from retired documentation workflows.

## Legacy Collections

- `design-proposals/`: historical design-proposal packages, indexes, snapshots, and acceptance records.
- `implementation-proposals/`: historical implementation and verification records formerly stored under `gameplay-alignment/`.

These collections preserve their bodies and embedded original paths as historical evidence. They are not gameplay or architecture authority, are not execution gates, and must not be used as templates for new work.

Do not read legacy bodies during normal bootstrap or current work. Read a specific record only when the user explicitly requests history, asks for an investigation that requires it, or resumes that exact subject. Current work starts from `work-items/active/`, `gameplay-design/`, `system-design/`, and repository state.
