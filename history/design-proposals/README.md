# Legacy Design Proposal Records

This directory preserves design-proposal records created under the retired current/expected-copy and archive lifecycle.

New design work does not create proposal packages, duplicate authority documents, relationship metadata, or proposal archives. Use the global two-stage workflow instead:

```text
discussion
-> user confirmation
-> thread execution baseline
-> update the relevant gameplay or system authority when the decision is durable
-> execution and verification
```

`gameplay-design/` and `system-design/` remain the only accepted design and architecture authority. Nothing in this directory authorizes implementation or overrides those documents.

Use `archived/README.md` for historical orientation. Read individual legacy bodies only when the user explicitly requests a record, resumes its exact subject, or asks for archive investigation.

Do not create new entries in this directory. Existing deferred material must re-enter normal discussion before any part of it is accepted or implemented.
