# Gameplay Alignment

This directory tracks the steady repair work needed to align the existing implementation and older documentation with the accepted gameplay and system-design authority.

It is not the place for final gameplay rules or architecture contracts.

```text
gameplay-design/     what the game should be
system-design/       how accepted systems should be structured
work-items/           confirmed active tasks and execution handoffs
gameplay-alignment/  what is currently out of alignment and how repair progresses
history/              legacy records, outside normal bootstrap
```

## Use

Use this directory for:

- gap tracking;
- durable workstream and acceptance records;
- migration workstream boundaries;
- acceptance checklists;
- old-document cleanup lists;
- repair sequencing.

Do not use it for:

- final product or gameplay rules;
- final architecture contracts;
- temporary daily notes;
- code-level implementation logs.
