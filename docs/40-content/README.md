# Content Index

This directory stores authored content specifications.

## Routes

- Character content: `characters/README.md`
- World content: `world/README.md`
- Tutorial content: `tutorial/README.md`

## Rule

Content overview documents should describe purpose, scope, and links. Detailed layouts, beat scripts, character definitions, encounter specs, and authored world setup should live in focused subdocuments.

Do not put product positioning, gameplay pillar definitions, or implementation contracts here. Use `../10-product/`, `../20-game-design/`, or `../30-technical-design/` for those layers.
