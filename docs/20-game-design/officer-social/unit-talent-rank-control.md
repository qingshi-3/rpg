# 单位天分、位阶与控制权

本文定义群像单位如何成长、晋升、承担职责和进入玩家直接控制。目标是支持大量角色和部下存在，同时避免全员重养成和全员手操造成疲劳。

## 核心原则

```text
天分决定成长上限。
位阶决定组织职责。
控制权决定玩家能否手操。
```

这三条轴不能混成一条。强单位不一定必须手操，不能手操的单位也不应失去价值。

## 天分

天分是单位的先天上限。初期设计中不可改变。

第一版建议分层：

| 天分 | 上限方向 | 说明 |
| --- | --- | --- |
| 凡才 | 老兵 / 精锐小队 | 可成长，但不成为完整角色。 |
| 良才 | 精英 | 可拥有一个核心特性或必杀机制。 |
| 英才 | 干部 / 小将 | 可带队、驻守、处理局部事件。 |
| 将才 | 区域总管 / 军团长 | 可承担区域性职责和较复杂自动决策。 |
| 传奇 | 亲控候补 / 核心席位 | 可竞争最高直接控制席位。 |

未来可以加入极稀有道具改变天分，但这会带来内容和模板扩展成本，不属于初期目标。

## 位阶

位阶是组织身份和职责，不等于战斗强度。

### 小兵

- 不手操。
- 使用最基础行为包，例如移动、攻击、跟随、守点、撤退。
- 可以升级，但受天分上限限制。
- 通常不进入完整关系和装备养成。

### 老兵 / 精锐

- 不手操。
- 比小兵有更高数值或一个简单特性。
- 仍以队伍或兵种为主，不要求玩家逐个养成。

### 精英个体

- 不手操。
- 有名字、特性、必杀或行为包。
- 可以显示 Intent 或蓄力计划。
- 玩家可以通过宏观指令提高释放精准度或改变倾向，但不能逐回合接管。

### 干部 / 将领

- 不默认手操。
- 可带队、驻守、执行委托、处理事件。
- 进入忠诚、野心、怨恨、职责适性和关系系统。
- 可以成为亲控候补或区域总管候补。

### 区域总管

- 管理一个区域内的多个低价值或中价值场域。
- 负责区域防务、巡逻、资源调配、小规模事件自动处理、招募训练和异常上报。
- 用于解决中后期地图扩大后城池低交互价值带来的管理疲劳。
- 初期不是重点，但系统设计应预留职责入口。

### 亲控席位

- 玩家可以完整手操的核心单位席位。
- 数量有限，但不必须固定为 4。
- 可随世界进度、权柄、建筑、契约或剧情条件增长。
- 名称可由世界观包装，例如四天王、七柱、亲卫、契约者等；系统层先称为 `DirectControlSlot`。

亲控席位不是单纯强度标签，而是玩家注意力和战术操作权的稀缺资源。

## 控制权

控制权分层建议：

| 控制权 | 玩家行为 | 单位行为 |
| --- | --- | --- |
| Direct | 完整选择移动、能力、目标。 | 按玩家命令行动。 |
| Commanded | 接受宏观指令。 | 自动选择具体行动。 |
| Autonomous | 玩家分配职责或区域。 | 按行为包、性格和职责自动行动。 |
| Simulated | 只参与世界层结算。 | 不进入战斗细节或只作为抽象战力。 |

宏观指令示例：

- 守点
- 集火
- 保命
- 等待充能
- 自动释放
- 保护目标
- 推进
- 撤退

非亲控单位的正确方向不是黑箱聪明 AI，而是可读计划和可解释结果。战斗内应复用 Intent / preview / BattleAction 边界，让玩家知道它准备做什么、能怎样影响它、最后为什么那样结算。

## 晋升

晋升条件应由通用数据表达：

```text
功绩 / 经验
+ 天分上限
+ 忠诚或契约稳定度
+ 空缺位阶或席位
+ 资源 / 仪式 / 任命成本
= 晋升结果
```

晋升主要解锁：

- 更高职责。
- 更强行为包。
- 更高带队能力。
- 少量特性或技能池。
- 是否能成为亲控候补。

晋升不应把每个单位都变成完整技能树、装备树和关系树。普通小兵可以成长，但不应要求玩家长期逐个维护。

## 亲控候补与模板

高天分单位进入亲控候补后，第一版应优先使用通用模板和技能池。

示例：

```text
DirectControlTemplate
  Role: striker / defender / support / controller
  AbilityPool
  PassiveTraitPool
  AIIntentFallback
  PromotionCosts
```

专属亲控机制可以留给少数关键角色。不要在初期要求每个高天分单位都有完整专属机制。

## 与关系系统的关系

位阶和控制权会改变关系。

示例：

- 提拔一名结义成员，其他结义成员可能因 `Rivalry`、`Obligation` 或 `Trust` 变化产生反应。
- 高野心干部长期不得晋升，可能忠诚下降。
- 高信任角色更愿意接受危险职责。
- 高怨恨角色在自动结算中更可能抗命、保守执行或借机离队。
- 区域总管的治理倾向会影响区域秩序、资源、冲突和召唤者关系。

这些反应仍然通过抽象 trait、relationship、bond 和 event 定义表达，不按具体人名硬编码。

## 与战斗系统的关系

已有战斗文档的三分法继续有效：

```text
Hero: directly controlled.
Minion: rule-driven.
Enemy: Intent-driven.
```

本设计将其扩展为：

```text
亲控单位 -> Ability-driven direct control
非亲控友军 -> Rule / Intent / behavior-package driven
敌人 -> Intent-driven
```

该段落是旧亲控/非亲控划分的设计参考，不是当前战斗架构入口。未来战斗行动应通过已接受的英雄带兵轻 RTS 架构定义命令、事件和结算边界；非亲控单位不应绕开战斗行动边界直接修改 HP、位置、死亡状态或长期战役状态。

## 设计风险

- 如果亲控席位无限开放，玩家会重新陷入全员操作疲劳。
- 如果所有高天分单位都需要专属亲控机制，内容工作量会快速失控。
- 如果非亲控单位计划不可读，玩家会认为自己被自动行为惩罚。
- 如果区域总管变成另一个重养成系统，中后期疲劳不会减少，只会换一种形式出现。

因此第一版应优先验证：

- 少量核心单位的战斗表现深度。
- 非亲控单位的 Intent / 行为包可读性。
- 小兵晋升受天分限制。
- 干部和区域总管通过职责产生世界价值，而不是通过全员手操产生价值。
