# 战略场域格子探索

本文定义 `WorldSite` 内格子式探索的玩家体验方向。实现合同见 `../../30-technical-design/world/strategic-world-site-grid-exploration.md`。

## 设计目标

场域探索解决代入感问题：玩家不是在菜单里抽象点击“潜入”或“强攻”，而是在场域地图上看到自己的小队位置、入口、哨岗、风险路线和目标点。

玩家应该理解：

```text
我从哪里进入。
我走到了哪里。
我为什么被发现。
我为什么在这个位置开战。
我清理过的地方如何变成后续经营资产。
```

## 核心循环

```text
大地图发现陌生或敌对场域
-> 进入场域探索态
-> 控制一个小队 marker 在格子上移动
-> 场域探索 tick 持续推进玩家小队和巡逻单位
-> 观察、调查、潜入、清理、强攻或撤离
-> 触发事件、战斗或解锁资源
-> 胜利或解决核心目标后转入经营态
-> 探索过的地点成为设施槽位、资源点、防御点或隐患
```

探索不是自由开放世界，也不是 RTS 群控。玩家控制的是“小队代表”，不是每个单位。

场域探索采用可暂停的离散实时制：玩家下达移动或调查意图后，系统按 `SiteExplorationTick` 推进所有探索 actor。玩家小队和巡逻单位都消耗探索行动力移动；到达目标、触发兴趣点、进入敌方警戒半径、请求战斗或玩家主动暂停时，探索推进停止并等待确认。

`SiteExplorationTick` 不是战斗回合。探索行动力不是战斗 AP，不进入 `TurnSystem`，也不影响战棋单位行动规则。

## 交互方式

- 左键可达格：小队移动到目标格。
- 左键探索点：小队先移动到交互范围，再打开行动。
- 点击移动后默认自动推进探索 tick，不需要玩家每格确认。
- 玩家可以暂停、改目标或撤离；风险触发时系统自动暂停。
- 右侧面板：显示当前探索点、风险、可执行行动和后果。
- 底部反馈：记录发现、警戒变化、资源获得和战斗触发原因。
- 探索 tick 只推进场域内模拟；明确耗时行动才推进 `WorldTick`。普通走路、快速观察和巡逻移动不推进世界时间。

## 场域状态

### 陌生 / 敌对场域

默认进入探索态。

玩家能看到：

- 入口区域。
- 小队当前位置。
- 已揭示道路。
- 可疑点、哨岗、资源痕迹或危险区。
- 警戒度。

玩家不能直接建造或经营。

### 已掌控场域

默认进入经营态。

玩家能看到：

- 设施槽位。
- 建筑状态。
- 驻军和工作人员。
- 产出、施工、修复和防御效果。

探索中揭示和解决过的点，会转化为经营态中的资产或隐患。

### 战斗态

只处理战棋战斗。

- 战斗回合不推进探索时间。
- AP 不用于探索移动。
- 战斗结束后通过结果回写场域。

## 探索点类型

推荐第一版探索点：

| 类型 | 玩家感受 | 常见行动 | 结果 |
|---|---|---|---|
| 入口 | 我从这里进入 | 观察、撤离、强攻 | 决定部署方向或撤退路线 |
| 哨岗 | 敌人可能发现我 | 观察、偷袭、绕行 | 降低伏击或触发警报 |
| 资源痕迹 | 这里有可利用物 | 调查、搜刮、清理 | 获得资源或解锁经营点 |
| 障碍 | 需要花时间处理 | 清理、爆破、绕行 | 消耗世界步并打开路线 |
| 核心营地 | 场域控制目标 | 潜入、强攻 | 触发战斗或占领 |

## 警戒度

警戒度让潜入和探索有过程感。

```text
0：敌人未察觉
1-2：轻微警觉
3-4：高警戒，危险行动更容易开战
5：警报，触发遭遇或强制撤离选择
```

行动必须说明可能后果，例如：

```text
潜入仓库
成功：获得物资并揭示内门
失败：警戒 +2，触发仓库遭遇战
```

玩家应感觉“是我的选择导致了风险”，而不是系统无缘无故开战。

第一版敌人发现规则使用固定警戒半径，不做视锥和遮挡判定：

```text
distance(playerCell, patrolCell) <= AlertRadiusCells
-> 暂停探索推进
-> 提示被发现原因
-> 请求战斗或进入后续 Alert 选择
```

巡逻单位可以移动，但第一版只沿作者配置的固定路线巡逻。不要做追击、搜索、听觉传播或随机发现检定；这些属于后续 AI 扩展。

## 战斗触发

战斗触发应绑定具体位置和行动。

示例：

```text
正门选择强攻 -> 正门部署战
仓库潜入失败 -> 仓库遭遇战
哨岗偷袭失败 -> 警报战
核心营地突袭 -> 主目标战
```

战斗开始时，探索状态应影响战斗：

- 当前格决定入口或部署侧。
- 警戒度决定敌人是否准备充分。
- 已解决哨岗可给我方突袭优势。
- 已清理障碍可打开战斗地图路线。

## 探索到经营的转换

探索胜利后，不应只是弹出奖励。场域要留下记忆：

```text
塌方矿道清理 -> 解锁矿场槽位
哨岗控制 -> 解锁防御塔或巡逻优势
仓库搜刮 -> 获得资源，后续作为储备点显示
核心营地胜利 -> 场域进入可经营状态
```

这样玩家会感觉自己经营的是刚刚亲自进入和改变过的场域。

## 第一版范围

第一版建议只做一个小场域切片，并允许使用 mock 内容验证长期架构：

```text
入口
破损矿车
哨岗
塌方矿道
内侧营地
```

必要体验：

- 小队在格子上移动。
- 1 到 2 个巡逻单位按固定路线移动。
- 玩家和巡逻单位都按探索 tick 消耗探索行动力。
- 玩家进入巡逻单位固定警戒半径后暂停并触发战斗入口。
- 点击点位打开行动。
- 行动改变警戒或点位状态。
- 至少一个行动触发战斗。
- 至少一个行动解锁经营资产。

暂不做：

- 多单位自由控制。
- 完整潜行视锥。
- 敌人追逐、搜索和复杂 AI。
- 物理碰撞探索。
- 大量搜刮物。
- 复杂随机潜入系统。
